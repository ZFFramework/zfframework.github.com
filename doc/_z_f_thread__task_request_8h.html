◆ ZFThreadTaskRequest()

zfidentity() ZFThreadTaskRequest ( const ZFListener &  taskCallback,
ZFObject *  taskUserData = (NULL),
ZFObject *  taskParam0 = (NULL),
ZFObject *  taskParam1 = (NULL),
ZFObject *  taskOwner = (NULL),
const ZFListener &  taskMergeCallback = (ZFThreadTaskRequestMergeCallbackDefault()) 
)

request a task that would be run on main thread after a proper time, safe to be called in background thread

task's param is the params passed to this method, and return value is ignored

tasks are ensured scheduled in different CPU time

if duplicated task is found, the mergeCallback would be called
two tasks are treated as duplicated task if task callback and owner are both the same, while param0/param1/userData won't be compared


the mergeCallback's param0 is a ZFThreadTaskRequestMergeCallbackData contains old and new task data (ZFThreadTaskRequestData), you should decide merged task and set it to ZFThreadTaskRequestMergeCallbackData::taskRequestDataMerged, whether retain existing one or allocating new one, or set it to null to show the task can not be merged
for example:
to ignore old task, you may retain new one and set it to taskRequestDataMerged
to ignore new task, you may retain the old one instead
to merge, allocate a new task data and setup necessary info
to disable merge, set taskRequestDataMerged to null, and old/new task would be scheduled separately
or, you may use the pre-defined callbacks such as ZFThreadTaskRequestMergeCallbackDoNotMerge
