◆ ZFFrameworkInit()

void ZFFrameworkInit ( void )
extern

must be called in main thread before any access of ZFFramework member

usually, this would been done by implementation of ZFFramework

ADVANCED:
if you want to dynamically load other libs written in ZFFramework after ZFFramework's main lib initialized:

  • you must manually call ZFFrameworkInit to initialize your newly loaded lib
  • when unload the additional lib, the ZF_GLOBAL_INITIALIZER_DESTROY's order is not ensured
  • ZFFrameworkInitFinishCallbacks and ZFFrameworkCleanupPrepareCallbacks won't be called when loading the additional lib
