Detailed Description

abstract page manager to manage multiple ZFUIPage

a page is an abstract object with those life cycle:

  • ZFUIPage::pageOnCreate, create necessary contents
  • ZFUIPage::pageOnResume, before making the page active
  • ZFUIPage::pageOnPause, before making the page deactive
  • ZFUIPage::pageOnDestroy, to destroy the page

and a page manager is an abstract object to manage multiple pages

pages is managed by life cycle request, you must use ZFUIPageManager's method to manage pages, including creating pages or showing pages
a page manager can hold many pages, however, only one page at most is resumed at one time, resuming another page would cause current page to be paused, pausing page manager would also causing current page to be paused
page's life cycle is requested by app code, and is ensured to be queued to run at future

typically, the page manager and page are not serializable, but it's recommended to make the page param serializable, so that pages can be created by reflection, to achieve advanced dynamic logic

ADVANCED:
typically, a page is a normal view to hold UI elements to interact with user, and a page manager is a native window or UI container to hold the pages, however, you may embeded a page manager to another container, even another page manager, for how to, please refer to embededCreate

◆ pageDelayDetachOnCheck()

virtual void ZFUIPageManager::pageDelayDetachOnCheck ( ZFUIPage *  page,
ZFUIPagePauseReasonEnum  reason 
)
protectedvirtual

check whether the page need delay to detach, do nothing by default

for some case, page need to process additional works after pause, and should not be detached immediately, to achieve this, you may override this method and call pageDelayDetachEnable, then finally notify finish by pageDelayDetachNotifyFinish to finally detach the page
pageDelayDetachEnable can be called multiple times, bust must be paired with pageDelayDetachNotifyFinish
all delay detach task must be finished before next attach task, otherwise, assert fail
delaying detach page would also cause pageDelayDestroyEnable to be set automatically

See also
pageDelayDestroyOnCheck

◆ pageAniOnUpdate()

virtual void ZFUIPageManager::pageAniOnUpdate ( ZFUIPage *  resumePageOrNull,
ZFUIPage *  pausePageOrNull,
ZFUIPageResumeReasonEnum  reason 
)
protectedvirtual

called to achieve actual page animation

by default, this method would:

  • if pageAniOverrideForOnce has been set, use the overrided animation
  • otherwise, use page's default animation setting

you may override without calling super to override default animation


the default impl:

ZFUIPage *pausePage = xxx; // page to pause
ZFUIPage *resumePage = xxx; // page to resume
resumePage->pageAniPriorityNeedHigher = zffalse;
pausePage->pageAniPriorityNeedHigher = zffalse;
pausePage->pageAniOnUpdate(resumePage, pausePage, reason);
resumePage->pageAniOnUpdate(resumePage, pausePage, reason);
zfbool pausePageHasHigherPriority = (!resumePage->pageAniPriorityNeedHigher && pausePage->pageAniPriorityNeedHigher);
pageManager->pageAniOnUpdatePriority(resumePage, pausePage, pausePageHasHigherPriority);

by default, pageAniOnUpdate would update page animation accorrding to page's resume and pause reason, and page requested by user would have higher animation priority (request resume or request destroy)
the page's animation:

  • ZFUIPage::pageAniResumeByRequest
  • ZFUIPage::pageAniResumeFromBackground
  • ZFUIPage::pageAniPauseToBackground
  • ZFUIPage::pageAniPauseBeforeDestroy

called to check whether needs to pageOnDetach page from manager

for managers that needs to keep more than one page being attached, you may simply override this method and return false
