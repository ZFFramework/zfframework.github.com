◆ ZFSerializableDataResolveCheckEnable

zfbool ZFSerializableDataResolveCheckEnable

whether enable resolve check during serialization, true by default

resolve check may consume more CPU but useful during develop to find ill-formed serializable data
when on, check would be performed and an error output would be printed, further more, assert fail if ZFFramework was built ad DEBUG mode
when off, no check would be performed, also, no resolve state would be saved, and all resolve related method would return false as resolved state

note, you must not change this config if some serialization logic is processing, usually, change it in app entry
