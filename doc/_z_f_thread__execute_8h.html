execute in main thread

automatically retain runnable's owner and params, and release them after finish
runnable won't be executed immediately even if current thread is main thread, it's queued instead, unless you call this function in main thread and set waitUntilDone to zftrue

note, sender would be retained until execute finish, while owner won't

See also
ZFThreadExecuteInNewThread, ZFThreadExecuteCancel
